# Google Sheets Integration Setup

This guide explains how to set up Google Sheets integration for the Brighter Mark Enterprises contact forms.

## Overview

The website contact forms now submit data directly to a Google Sheet via Google Apps Script. This replaces the previous email-based submission system.

## Setup Steps

### 1. Create a Google Sheet
1. Go to [Google Sheets](https://sheets.google.com)
2. Create a new spreadsheet named "Brighter Mark Contact Submissions"
3. Set up the following columns in the first row (Sheet1):
   - **Timestamp** (auto-generated by script)
   - **Business Name**
   - **Contact Person**
   - **Phone Number**
   - **Email Address**
   - **Service Type**
   - **Property Type**
   - **Square Footage**
   - **Cleaning Frequency**
   - **Message / Special Requirements**
   - **Page URL**
   - **Submission Source**

### 2. Create Google Apps Script
1. In your Google Sheet, click **Extensions** → **Apps Script**
2. Delete any default code and paste the following script:

```javascript
function doPost(e) {
  try {
    // Parse the incoming data
    const params = e.parameter;
    
    // Get the active spreadsheet
    const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
    
    // Prepare row data
    const timestamp = new Date().toLocaleString();
    const rowData = [
      timestamp,
      params.business_name || '',
      params.contact_person || '',
      params.phone || '',
      params.email || '',
      params.service_type || '',
      params.property_type || '',
      params.square_footage || '',
      params.frequency || '',
      params.message || '',
      params.page_url || '',
      params.source || 'brighter-mark-website'
    ];
    
    // Append the data to the sheet
    sheet.appendRow(rowData);
    
    // Return success response
    return ContentService
      .createTextOutput(JSON.stringify({
        result: 'success',
        message: 'Data saved to Google Sheet',
        timestamp: timestamp
      }))
      .setMimeType(ContentService.MimeType.JSON);
      
  } catch (error) {
    // Return error response
    return ContentService
      .createTextOutput(JSON.stringify({
        result: 'error',
        message: error.toString()
      }))
      .setMimeType(ContentService.MimeType.JSON);
  }
}

// For testing: function to handle GET requests
function doGet(e) {
  return ContentService
    .createTextOutput(JSON.stringify({
      status: 'online',
      message: 'Google Sheets API is ready to receive form submissions'
    }))
    .setMimeType(ContentService.MimeType.JSON);
}
```

### 3. Deploy the Script as Web App
1. Click **Deploy** → **New deployment**
2. Select type: **Web app**
3. Configuration:
   - **Description**: "Brighter Mark Contact Form Handler"
   - **Execute as**: "Me" (your Google account)
   - **Who has access**: "Anyone" (important for public forms)
4. Click **Deploy**
5. Copy the **Web app URL** (looks like: `https://script.google.com/macros/s/ABC123xyz/exec`)

### 4. Update Website Configuration
1. Open `js/contact-form.js`
2. Find the `API_CONFIG` section (around line 5-9)
3. Replace the placeholder URL with your actual Google Apps Script URL:
   ```javascript
   const API_CONFIG = {
     ENDPOINT: 'https://script.google.com/macros/s/YOUR_ACTUAL_SCRIPT_ID/exec',
     METHOD: 'POST'
   };
   ```

### 5. Test the Integration
1. Open your website's contact page (`contact.html`)
2. Fill out the form with test data
3. Submit the form
4. Check your Google Sheet - a new row should appear with the submitted data

## Troubleshooting

### Common Issues

1. **"Script URL not found" error**
   - Ensure the script is deployed as a web app
   - Verify the URL is correct (case-sensitive)

2. **No data appears in Google Sheet**
   - Check the script execution logs (View → Logs in Apps Script)
   - Verify the sheet name matches in the script
   - Ensure "Anyone" has access to the web app

3. **CORS errors in browser console**
   - The current implementation uses `mode: 'no-cors'` to avoid CORS issues
   - This is normal and expected for Google Apps Script

4. **Form submits but shows error message**
   - The form shows success even with `no-cors` mode
   - Check Google Sheet for actual data submission
   - Review browser console for any network errors

## Data Flow

1. User submits form on website
2. JavaScript collects form data and sends to Google Apps Script URL
3. Google Apps Script receives data via `doPost()` function
4. Script appends data to Google Sheet
5. User sees success message on website

## Security Considerations

- The web app is publicly accessible (required for form submissions)
- Consider adding basic validation or spam protection in the Apps Script
- No sensitive data (passwords, API keys) should be included in form submissions
- Regular monitoring of the Google Sheet is recommended

## Maintenance

- Periodically check the Google Sheet for new submissions
- The script handles all form submissions from both contact pages
- If you need to modify the data structure, update both the script and the sheet columns
- Keep a backup of the Google Sheet data
